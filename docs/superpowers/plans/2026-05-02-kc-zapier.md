# kc-zapier Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Zapier integration to KonaClaw. Zapier already exposes its catalog as a remote MCP server, so the bulk of this is **configuration + reuse of kc-mcp's dynamic-install pattern** rather than fresh code. Each new zap that an agent uses gets the destructive-tier-by-default treatment that's already in place for MCP tools.

**Architecture:** A small Python package `kc_zapier` that depends on `kc-mcp`. Provides:
- `register_zapier_mcp(manager, api_key)` — convenience to register the Zapier MCP server with `kc_mcp.MCPManager`.
- A `find_or_install_zap` agent meta-tool that takes a natural-language description, queries the user's Zapier account for matching zaps, and either invokes an existing one or surfaces a "create zap in Zapier first, then link it" prompt.
- A `kc_zapier.config` loader that picks up the Zapier API key from `kc_connectors.secrets.load_secrets()` (the encrypted store lands in kc-supervisor v0.2).

**Tech Stack:** Python 3.11+. Reuses everything from kc-mcp.

**Repo bootstrap:** `~/Desktop/claudeCode/SammyClaw/kc-zapier/`.

**Scope decisions:**
- ✅ Wrap Zapier MCP through kc-mcp.
- ✅ Per-zap tier defaults to **destructive** until user downgrades.
- ⏸ In-dashboard zap discovery UI ("create zap in Zapier" deep-link) — deferred to v0.2.

# src/kc_zapier/config.py
from __future__ import annotations
from dataclasses import dataclass
from kc_connectors.secrets import load_secrets


@dataclass
class ZapierConfig:
    api_key: str
    mcp_url: str = "https://mcp.zapier.com/api/mcp/mcp"


def load_config() -> ZapierConfig:
    secrets = load_secrets()
    if "zapier_api_key" not in secrets:
        raise KeyError("zapier_api_key missing from secrets.yaml")
    return ZapierConfig(api_key=secrets["zapier_api_key"])
```

# src/kc_zapier/register.py
from __future__ import annotations
from kc_mcp.handle import MCPServerHandle
from kc_mcp.manager import MCPManager
from kc_zapier.config import ZapierConfig


class _HTTPMCPHandle(MCPServerHandle):
    """MCP server reached over HTTP rather than stdio."""

    def __init__(self, *, name: str, url: str, headers: dict[str, str]) -> None:
        super().__init__(name=name, command="<http>", args=[], env={})
        self.url = url
        self.headers = headers

    async def start(self) -> None:
        from mcp import ClientSession
        from mcp.client.sse import sse_client  # SSE transport in the official SDK
        self._cm = sse_client(self.url, headers=self.headers)
        read, write = await self._cm.__aenter__()
        self._session_cm = ClientSession(read, write)
        self._client = await self._session_cm.__aenter__()
        await self._client.initialize()


async def register_zapier_mcp(manager: MCPManager, cfg: ZapierConfig) -> None:
    handle = _HTTPMCPHandle(
        name="zapier",
        url=cfg.mcp_url,
        headers={"Authorization": f"Bearer {cfg.api_key}"},
    )
    await manager.register_handle(handle)
```

## Task 3: find_or_install_zap Meta-Tool

**Files:**
- Create: `src/kc_zapier/meta_tool.py`
- Test: `tests/test_meta_tool.py`

**Why:** Convenience for the agent: when the user says "set up X in Zapier," the agent calls this meta-tool with a description. The tool searches the registered `mcp.zapier.*` tools for matches by name/description; if found, it returns the tool name so the agent can call it directly. If not, it returns a structured response telling the agent to ask the user to create the zap in Zapier first (with a deep-link URL). No silent zap creation — the user always sets up zaps in Zapier's own UI.

```python
# src/kc_zapier/meta_tool.py
from __future__ import annotations
from kc_core.tools import Tool
from kc_mcp.manager import MCPManager


def build_find_or_install_zap_tool(*, manager: MCPManager) -> Tool:

    async def impl(description: str) -> str:
        terms = [t.lower() for t in description.split() if len(t) > 2]
        candidates = []
        for tool in manager.all_tools():
            if not tool.name.startswith("mcp.zapier."):
                continue
            text = f"{tool.name} {tool.description}".lower()
            score = sum(1 for t in terms if t in text)
            if score > 0:
                candidates.append((score, tool.name, tool.description))
        candidates.sort(reverse=True)

        if candidates:
            top = candidates[:5]
            lines = ["Existing zap tools that look relevant — call one directly:"]
            for _, name, desc in top:
                lines.append(f"- {name} — {desc[:120]}")
            return "\n".join(lines)

        return (
            "No matching zap is connected. Ask the user to create one in Zapier first, "
            "then re-link the account. Zapier dashboard: https://zapier.com/app/dashboard"
        )

    return Tool(
        name="find_or_install_zap",
        description=(
            "Find a Zapier zap matching the description, or tell the user how to create one. "
            "This does NOT create zaps — it only searches what's already connected."
        ),
        parameters={
            "type": "object",
            "properties": {"description": {"type": "string"}},
            "required": ["description"],
        },
        impl=impl,
    )
```
